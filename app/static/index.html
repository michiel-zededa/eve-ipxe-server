Option B — Configure your existing DHCP server

        
ISC DHCP / dnsmasq / isc-kea pointing to this server

        

          
isc-dhcpd.conf

          
next-server SERVER_HOST;      # TFTP server
filename "undionly.kpxe";      # for BIOS PXE clients

# UEFI clients:
if substring (option vendor-class-identifier, 0, 20) = "PXEClient:Arch:00007" {
  filename "ipxe.efi";         # x86_64 UEFI
} elsif substring (option vendor-class-identifier, 0, 20) = "PXEClient:Arch:00011" {
  filename "ipxe-arm64.efi";   # ARM64 UEFI
}

# Once iPXE is running, send it straight to the boot script:
if exists user-class and option user-class = "iPXE" {
  filename "http://SERVER_HOST:WEBUI_PORT/ipxe/boot.ipxe";
}

UEFI Arm64 (Raspberry Pi / Jetson)

        
Prerequisites for ARM64 UEFI network boot

        

          ℹ
          

            
UEFI firmware required

            

              ARM64 network boot requires UEFI firmware on the device.
              


              Raspberry Pi 4: Install the Pi 4 UEFI firmware on an SD card, then PXE-boot from it.
              

              NVIDIA Jetson: Jetson Orin with JetPack 5/6 supports UEFI. Use the nvidia-jp5 or nvidia-jp6 variant for the raw installer.
              


              DHCP option 93 (client-arch=11) identifies ARM64 UEFI clients; dnsmasq and ISC-DHCP both support it.

        ►
        Already have a DHCP server? Configure it to point clients here instead
      
      

        

          Add the following PXE boot options to your existing DHCP server.
          Replace SERVER_HOST with this server's LAN IP
          (—).
        


        
dnsmasq

        

          
dnsmasq.conf

          
# TFTP server address (option 66)
dhcp-option=66,SERVER_HOST

# BIOS PXE clients
dhcp-boot=undionly.kpxe,SERVER_HOST,SERVER_HOST

# UEFI x86_64 clients (arch 7)
dhcp-match=set:efi-x86_64,option:client-arch,7
dhcp-boot=tag:efi-x86_64,ipxe.efi,SERVER_HOST,SERVER_HOST

# UEFI arm64 clients (arch 11)
dhcp-match=set:efi-arm64,option:client-arch,11
dhcp-boot=tag:efi-arm64,ipxe-arm64.efi,SERVER_HOST,SERVER_HOST

# Once iPXE is running, redirect to the boot script
dhcp-userclass=set:ipxe,iPXE
dhcp-boot=tag:ipxe,http://SERVER_HOST:WEBUI_PORT/ipxe/boot.ipxe

        


        
ISC DHCP (dhcpd.conf)

        

          
dhcpd.conf

          
next-server SERVER_HOST;
filename "undionly.kpxe";

# UEFI clients override the filename
if substring (option vendor-class-identifier, 0, 20) = "PXEClient:Arch:00007" {
  filename "ipxe.efi";         # x86_64 UEFI
} elsif substring (option vendor-class-identifier, 0, 20) = "PXEClient:Arch:00011" {
  filename "ipxe-arm64.efi";   # ARM64 UEFI
}

# Once iPXE is running, send it to the boot script
if exists user-class and option user-class = "iPXE" {
  filename "http://SERVER_HOST:WEBUI_PORT/ipxe/boot.ipxe";
}
