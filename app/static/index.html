Option A — dnsmasq (bundled, recommended for lab)

        
Start the optional dnsmasq DHCP service included in this stack

        

          
shell

          
# Enable DHCP on your network interface
docker compose --profile dnsmasq up -d

# Tune in .env before starting:
# INTERFACE=eth0
# DHCP_RANGE=192.168.1.100,192.168.1.200,12h
# DHCP_ROUTER=192.168.1.1
# SERVER_HOST=192.168.1.10

        

      


      

        
Option B — Configure your existing DHCP server

        
ISC DHCP / dnsmasq / isc-kea pointing to this server

        

          
isc-dhcpd.conf

          
next-server SERVER_HOST;      # TFTP server
filename "undionly.kpxe";      # for BIOS PXE clients

# UEFI clients:
if substring (option vendor-class-identifier, 0, 20) = "PXEClient:Arch:00007" {
  filename "ipxe.efi";         # x86_64 UEFI
} elsif substring (option vendor-class-identifier, 0, 20) = "PXEClient:Arch:00011" {
  filename "ipxe-arm64.efi";   # ARM64 UEFI
}

# Once iPXE is running, send it straight to the boot script:
if exists user-class and option user-class = "iPXE" {
  filename "http://SERVER_HOST:WEBUI_PORT/ipxe/boot.ipxe";
}

        

      


      

        
Option C — Quick test with QEMU

        
Boot a VM directly from this server (no DHCP needed)

        

          
shell

          
qemu-system-x86_64 \
  -m 4G -smp 2 \
  -enable-kvm \
  -drive format=raw,file=/tmp/eve-test.img,if=virtio \
  -netdev user,id=net0,tftp=/path/to/tftp-root,bootfile=/boot.ipxe \
  -device virtio-net,netdev=net0 \
  -nographic

        

      


      

        
UEFI Arm64 (Raspberry Pi / Jetson)

        
Prerequisites for ARM64 UEFI network boot

        

          ℹ
          

            
UEFI firmware required

            

              ARM64 network boot requires UEFI firmware on the device.
              


              Raspberry Pi 4/5: Install the Pi 4 UEFI firmware on an SD card, then PXE-boot from it.
              

              NVIDIA Jetson: Jetson Orin with JetPack 5/6 supports UEFI. Use the nvidia-jp5 or nvidia-jp6 variant for the raw installer.
              


              DHCP option 93 (client-arch=11) identifies ARM64 UEFI clients; dnsmasq and ISC-DHCP both support it.
